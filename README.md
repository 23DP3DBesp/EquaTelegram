# Bass Bot — обновлённая версия

## Установка

```bash
npm install
```

Проверь `.env` — токен бота уже перенесён из твоего файла, добавлены новые переменные:

```
BOT_TOKEN=...
MAX_TRACKS=200            # лимит треков в библиотеке
MAX_FILE_SIZE_MB=25       # лимит размера присылаемого файла
KEEP_PROCESSED_VERSIONS=3 # сколько обработанных копий трека хранить
PORT=3000
```

## Запуск (как раньше, два процесса)

```bash
npm start   # сервер Mini App (server.js)
npm run bot # телеграм-бот (index.js), во втором терминале
```

## Что нового

**Обработка звука** (`audioProcessor.js`, `presets.js`)
- Многополосный компрессор (низ/середина/верх обрабатываются раздельно)
- Стерео-расширение
- De-esser
- Пресеты "Для наушников" и "Для колонок машины"
- Нормализация под Spotify / YouTube / клубный звук
- Fade-in/fade-out, обрезка тишины в начале/конце

**Библиотека** (`db.js`, `server.js`, вкладка "Библиотека")
- Плейлисты, теги, поиск, сортировка
- История прослушиваний (вкладка "История")
- Групповая обработка (применить пресет сразу к нескольким трекам)
- Автоочистка старых обработанных версий (`POST /api/cleanup`, вызывается автоматически при каждом рендере)

**Интерфейс** (`public/index.html`)
- A/B сравнение оригинал/обработанный одной кнопкой
- Мини-плеер снизу экрана с прогресс-баром — виден на любой вкладке
- 3 темы (красная/синяя/светлая), переключение вверху страницы
- Спектр-визуализатор в реальном времени
- Waveform-превью трека
- Долгое нажатие на точку графика — сброс этой полосы в 0
- Системные (нельзя удалить) пресеты отдельно от твоих кастомных
- Крупные кнопки-пресеты на главном экране
- Автовключение последнего использованного пресета при открытии

**Соцфункции**
- Экспорт пресета в код (кнопка "Код") + импорт по коду — делишься пресетом текстом
- Экспорт готового трека — кнопка "Сделать файл" рендерит трек через ffmpeg со всеми текущими настройками и даёт ссылку на скачивание
- "Топ пресетов" — сортировка по количеству добавлений в избранное (в рамках вашей библиотеки; у бота нет разделения по пользователям, поэтому полноценной публичной статистики между разными людьми нет)

**Инфраструктура**
- `logger.js` — логи пишутся в `logs/app.log`
- `backup.js` — бэкап `bassbot.db` в `backups/`, можно повесить на cron (пример команды — в самом файле)
- `Dockerfile` + `docker-compose.yml` — контейнеризация сервера и бота
- Лимиты на размер файла и количество треков (`config.js`)

## Что осталось за скобками и почему

- **Голосовое управление** — не делал по твоей просьбе.
- **Автоопределение жанра** — требует отдельную ML-модель распознавания музыки (например, стороннее API), это не доработка вечера, а отдельная задача.
- **Деплой на VPS** — Dockerfile и docker-compose готовы, но зайти на твой сервер и выполнить команды я не могу. Когда будет сервер — подскажу конкретные шаги.
- **"Топ пресетов сообщества"** — в текущей архитектуре бота нет разделения пользователей (все, кто пишет боту, делят одну общую библиотеку и БД), так что полноценная социальная механика потребует сначала добавить систему пользователей — отдельная большая задача.

## Важно про безопасность

Токен бота лежал в `.env`, который ты прислал мне напрямую — сам файл никуда, кроме этого чата, не попал, но если репозиторий когда-либо был публичным или токен куда-то засветился, лучше перевыпустить его через @BotFather (`/token`).
